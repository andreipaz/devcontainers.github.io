O que são Containers de Desenvolvimento?

Um contêiner de desenvolvimento (dev container, pra encurtar) permite que você use um contêiner como um ambiente de desenvolvimento completo. Pode ser usado pra rodar uma aplicação, para separar ferramentas, bibliotecas, ou runtimes necessárias para trabalhar com uma base de código, e para ajudar com testes e integração contínua. Dev containers podem ser rodados local ou remotamente, privados ou numa cloud pública, em uma variedade de ferramentas e editores suportados.

As Especificação de Dev Container buscam formas de enriquecer formatos já existentes com parametrizações específicas para desenvolvimento, ferramentas e configurações enquanto ainda possibilita a opção de um único container, não orquestrado – para que eles possam então serem utilizados como ambientes de codificação ou para integração contínua e testes. Para além dos principais metadados, a spec também permite que desenvolvedores utilizem Features e Templates para compartilhar e reutilizar etapas de configuração.

Use ou crie definições de dev containers para múltiplas stacks e ferramentas.
